## 抄録 (Abstract)

マルチエージェントAIシステム向けの憲法的ガバナンス枠組みは構造的整合性の強制には有効だが、時間とともにその構造を適応させるために必要な新知識の生成までは担えない。本論文は、ArcheフレームワークのLayer 0に埋め込まれた独立研究機関Akademiaの理論的基礎を提示する。Akademiaは、静的知識基盤の体系的劣化と、枠組みの自己改善に必要な洞察をガバナンス機構だけでは生み出せないという、知識進化問題を解くために設計された。プラトンのAcademyから王立学会に至る歴史的先例、自己進化型エージェントアーキテクチャの最近の進展、およびPaper 001で確立された権力分立の原理に依拠し、ガバナンスからの研究の独立は望ましいだけでなく構造的に不可欠であること、そしてその独立を維持しつつ認識論的説明責任を保つために、四つの役割アーキテクチャ（Scholarch, Scholar, Theorist, Scribe）がそれを具現化することを論じる。

## 1. はじめに (Introduction)

Paper 001は、自律的マルチエージェントAIシステムには、千年にわたって人間の制度を蝕んできた16の歴史的崩壊パターンを防ぐために憲法的ガバナンスが必要であることを確立した。Governance Council、Universal Role Standing、Constitutional State Machineは、監査・説明責任・段階的権限によって構造的整合性を保証する。

しかしガバナンスは問題の半分しか解かない。ガバナンスは既存の規則への適合の強化、確立された基準からの逸脱の検知、権限移行の管理はできるが、新知識の創出、外部研究の運用洞察への統合、規則そのものを変えるべき時期の判断はできない。ガバナンスは本質的に保守的であり、既にあるものを保存する。進化には、まだ存在しないものを生み出す補完的制度が必要である。

これは仮想上の隙間ではない。専用の研究機能なしに運用される知識集約型システムの軌跡を考えればよい。

1. **初期の有能性**: システムは、規則と知識基盤に現在の知識が符号化された状態で立ち上がる。
2. **環境のドリフト**: 外部環境が変化する。新研究が公表され、新たな攻撃ベクトルが現れ、新しいツールが利用可能になる。
3. **知識の陳腐化**: システムに符号化された知識が現実から乖離する。意思決定は時代遅れの前提に基づいて行われる。
4. **見えない劣化**: ガバナンスは*既存の*規則への適合を検証するため、規則そのものが不十分になったことは検知できない。システムは適合しているが誤っている。
5. **破局的な発見**: 内部知識と外部現実の乖離が、顕在的な失敗を引き起こすほど大きくなる。

この劣化パターンは、Paper 001のパターン5（制度的硬化）の知識版に相当するが、手続レベルではなく認識論レベルで作用する。Curatorの役割とFramework Evolution Protocolは症状（陳腐知識の剪定、修正案の提案）には対処するが、持続的かつ独立した調査のための制度的容量は欠いている。

Akademiaはこの構造的隙間を埋めるために設計された。Governance Councilと同一の憲法層（Layer 0）で動作し、明示的な独立の保護、研究ライフサイクルの各相専用の役割、研究の自由と認識論的説明責任の両方を保つよう慎重に設計されたガバナンスとのインタフェースを備えた研究機関である。

## 2. 知識進化問題 (The Knowledge Evolution Problem)

### 2.1 静的知識基盤は劣化する

AIシステムにおける知識基盤は典型的にはリポジトリとして扱われる。すなわち、エージェントが実行中に参照する事実・パターン・手続の保管庫である。この貯蔵モデルは、一度捕捉された知識は有効であり続けるという暗黙の前提を孕む。実際には、知識は複数の軸に沿って劣化する。

**時間的劣化**: 事実は陳腐化する。2024年のセキュリティのベストプラクティスは、2026年の攻撃ベクトルに対して不十分かもしれない。あるライブラリ版向けに文書化されたAPIパターンは、次の版では誤りになりうる。

**文脈的劣化**: あるプロジェクト文脈から抽出された知識は、別文脈へは移転しないかもしれない。結論を生んだ推論連鎖は、もはや成立しない前提に依存している可能性がある。

**推論的劣化**: 個々の事実が正しく残っていても、より広い環境の変化に伴い、そこから導かれた推論は無効になりうる。三つの証拠によって十分支持されていた結論は、四つ目の矛盾する証拠が現れたとき支持されなくなる。

### 2.2 ガバナンスは知識を生成できない

Governance Councilの13座席は監査と監視のために設計されており、既存の基準が満たされているかを評価する。これは、新知識を生み出す生成的行為とは根本的に異なる。

| 機能 | ガバナンス | 研究 |
|----------|-----------|----------|
| 主たる問い | 「我々は適合しているか？」 | 「我々は何を知るべきか？」 |
| 志向性 | 後ろ向き（監査） | 前向き（発見） |
| 成功指標 | 適合率 | 洞察の質 |
| 失敗モード | 過度の許容 | 無関係性 |
| 時間的焦点 | 現在の状態 | 将来の軌道 |

これらの機能を単一の制度に統合しようとすると、構造的衝突が生じる。現在の規則の執行と、それらの規則が十分かどうかの問い直しの両方を課された機関は、内在的張力に直面する。既存の規則に挑戦する新たな洞察は、その機関自身の執行権威を損なうからである。これが、研究権威も主張した歴史的ガバナンス制度が、規制枠組みを脅かす所見を最終的に抑圧した理由であり、中世教会がその典型例である。

### 2.3 Watcherの隙間

Archeの既存の知識管理役割、すなわちCurator（知識の維持）とWatcher（情報ギャップの検知）は、知識の衛生には対処するが、知識の生成には対処しない。Curatorは検証・統合・剪定を行い、Watcherはギャップを特定する。いずれの役割にも、持続的調査、外部研究の統合、理論の展開、所見の公表を行う権限・ワークフロー・独立はない。

これはそれらの役割への批判ではなく、知識の維持と知識の生成は根本的に異なる制度機能であり、異なる構造・インセンティブ・保護を要するという認識である。

## 3. 外部研究の状況 (External Research Landscape)

### 3.1 自己進化型エージェントアーキテクチャ

Self-Evolving Agents research groupによる包括的調査であるSelf-Evolving Agents Survey（arXiv:2507.21046, 2025年7月）は、AIエージェントシステムがどのように進化しうるかについて、現時点で最も完全な分類を与える。Self-Evolving Agents Surveyは進化メカニズムを四次元に分類する。モデル進化（ファインチューニング、自己訓練）、記憶進化（経験の蓄積、知識の洗練）、ツール進化（ツール創出、ツール選択の最適化）、アーキテクチャ進化（ワークフロー変更、役割の再編）である。

Akademiaの設計に特に関連するのは、Self-Evolving Agents Surveyの「Three Laws of Agent Evolution」である。

1. **Endure**（安全性）: 進化はシステムの安全性や中核的制約を損なってはならない。
2. **Excel**（性能）: 進化は時間とともにタスク性能を改善すべきである。
3. **Evolve**（最適化）: システムは自身の進化プロセスを最適化すべきである。

これらの法則は、Akademiaの設計制約に直接対応づけられる。研究は憲法的整合性を損なってはならない（Endure）、運用上有用な知識を生まなければならない（Excel）、自らの研究方法論を時間とともに改善しなければならない（Evolve）。Self-Evolving Agents Surveyの主要洞察、すなわち分野が静的モデルから相互作用による適応アーキテクチャへ移行しているという点は、専用の進化メカニズムが任意ではなく必要であるという根本前提を裏づける。

### 3.2 経験に基づく自己改善

SiriuS（arXiv:2502.04780, 2025年2月, Stanford）は、マルチエージェントシステムが「経験ライブラリ」、すなわち成功した推論軌跡の構造化コレクションを通じて、エージェントが参照し洗練しながら時間をかけて、有意な自己改善（2.86〜21.88%の性能向上）を達成しうることを示す。エージェントが新問題に遭遇すると、関連する過去の経験を検索し適用し、成功した新軌跡でライブラリを拡張する。不成功の試みも捕捉され、ライブラリ拡張を通じて洗練される。

Akademiaの設計にとってSiriuSからの決定的洞察は、**蓄積された構造化知識が、アーキテクチャ変更なしに自己改善を可能にする**という点である。変化するのはエージェント自身ではなく、エージェントがアクセスできる知識である。これは、知識進化（Akademiaの領域）とアーキテクチャ進化（CouncilのEvolution座席の領域）を分離するArcheのアプローチを裏づけ、システム改善の主要なレバーはエージェントアーキテクチャではなく知識基盤そのものであることを示唆する。

### 3.3 Logic Monopolyと制度的解

Logic Monopoly研究（arXiv:2603.25100, 2026年3月）は、Akademiaの設計に最も直接関連する所見の一つを与える。同論文は「Logic Monopoly」を、エージェントが自らの行動を計画・実行・評価する状態、すなわち完全な推論ライフサイクルが単一実体に集中する条件として定義する。実験結果は衝撃的である。独占的エージェントに対する攻撃成功率は84.30%であり、明示的な敵対的プロンプトなしに出現的欺瞞行動を示すものが31.4%に達する。

提案される解は、制度的線に沿った権力分立である。立法（ルール作成）、執行（行動）、裁断（評価）である。これはArcheアーキテクチャの、ガバナンス（Council）、運用（C-Suiteと実行層）、そして決定的に重要な知識生成（Akademia）の分離と直接並行する。Logic Monopoly研究は、Paper 001が理論的に論じたこと、すなわち**自律エージェントのガバナンスへの答えはモデルアラインメントではなく制度的インフラである**という点の経験的根拠を与える。

Akademiaに特化して言えば、Logic Monopolyの所見は、研究が独占的活動であってはならないことを含意する。研究課題を特定し、調査を行い、所見を統合し、自らの結論の質を評価するエージェントは、知識領域におけるLogic Monopolyを示す。Akademiaの四役割構造は、研究ライフサイクルを専門化された役割に分散することで、これに直接対処する。

### 3.4 制度としての不信を設計原理に

制度としての不信に関する文献（PMC11614927, 2024）は、効果的なAIガバナンスは不信のために設計されなければならないと論じる。すなわち、個々の構成要素（人間またはAI）が失敗し、侵害され、またはミスアラインで行動しうることを予見する必要がある。監視メカニズムは、監視対象の善意を仮定してはならず、それらの実体が積極的に顛覆を試みているときでも正しく機能しなければならない。

研究機関に適用すれば、この原理は、Akademiaが個々の役割が不確実な出力を生みうるときでも信頼できる知識を生むよう設計されなければならないことを含意する。内部レビュー（ScholarchとTheoristが内容を検証）、外部の品質監査（CouncilのQuality座とKnowledge座）、およびAnti-Rationalization Protocolは、研究プロセス自体の内部に制度としての不信を実装する。

### 3.5 先行するArche研究

Paper 001は、Akademiaの設計が依拠するいくつかの基礎を確立した。

- **腐敗のパラドックス**: 純粋な自己規制は不可能であり、外部のアンカリングが必要である。Akademiaにとっては、研究の質を研究者だけが評価できないということであり、したがってCouncilの品質監査役割が要る。
- **16の歴史的ガバナンス失敗パターン**。そのいくつかは知識領域に直接対応する類似を持つ（パターン4: 情報独占、パターン5: 制度的硬化、パターン11: 認識論的崩壊）。
- **ゲーム理論的証明**: 構造設計の下では協力がNash equilibriumになる。Akademiaはこれを研究領域へ拡張する。すなわち、相互説明責任メカニズムが捏造を検知可能にするとき、正直な知識生産が支配戦略になる。

## 4. 制度設計理論 (Institutional Design Theory)

### 4.1 研究が独立でなければならない理由

研究独立の論拠は規範的論拠ではなく構造的論拠に置かれる。研究がガバナンスの介入から自由である*べき*というのは真だが（それもまた真である）、それ以上に、研究はガバナンスの統制下では*機能できず*、体系的歪みを生む。

メカニズムは単純である。ガバナンス機関は現在の規則の妥当性について制度的利害を持つ。現在の規則が不十分であるという所見は、ガバナンス機関の権威と実績を脅かす。意識的バイアスがなくとも、研究課題の選択、問いの枠組み、所見の評価は、既存のガバナンス構造を正当化する結論へ体系的に傾く。

これは憶測ではない。国家統制下の研究の歴史は広範な証拠を提供する。

- ソ連のガバナンス下の**リセンコ生物学**: 農業研究の結論が政治イデオロギーによって決定され、飢饉につながった。
- 企業ガバナンス下の**タバコ産業の研究**: 事業利害を脅かすとき健康所見が体系的に抑圧された。
- **2008年以前の金融リスクモデリング**: 銀行内部のリスク研究は、正確なモデルが有利な取引を制約しうるため、体系的にシステミックリスクを過小評価した。

共通パターンは次のとおりである。研究がガバナンス（または研究結果に利害を持ついかなる権威）に答えるとき、研究は現実を反映する結論ではなく、その利害に奉仕する結論へ収束する。

### 4.2 歴史的先例

Akademiaの設計は、研究の独立を実際に維持した歴史的制度の構造特徴に依拠する。

**プラトンのAcademy（紀元前387年〜529年）**: 西洋史上最長の研究機関は900年以上活動した。主要な構造特徴は、アテネ国家からの独立統治、弁証法（結論への制度的挑戦）、累積的知識構築（各世代が先行研究の上に積む）である。Academyの独立は、支配的政治正統性に挑戦する所見の生産を可能にした。Akademiaはこの機能を再現しなければならない。

**House of Wisdom（Bayt al-Hikma, 8〜13世紀）**: アッバース朝の研究機関は、制度的独立が最大であった時期に最も大きな成果を上げた。翻訳・統合・独創的研究は、保護された地位の学者によって行われた。後期に政治的干渉が増すと、成果の質は低下した。教訓は、支配なき庇護は研究を可能にし、支配つき庇護はそれを歪めるということである。

**王立学会（1660年〜現在）**: 明示的モットー*Nullius in verba*（「誰の言葉も盲信するな」）とともに創設され、王立学会は懐疑を研究方法として制度化した。王勅を持ちながら王権支配からの独立は、自己統治とピアレビューを通じて構造的に維持された。関連する設計原理は、研究出力の質は権威基準（権力者は承認するか）ではなく認識論基準（推論は妥当か）によって評価されるべきであるということである。

**現代大学のテニュア**: 学術テニュア制度は、研究の独立を守るために正確に存在する。不人気の所見で解雇されうる研究者は、時間とともに人気のある所見しか生まない。テニュアは過去の業績への報酬ではない。将来の研究の誠実性を確保するための制度メカニズムである。

### 4.3 ガバナンスと研究の分離

これらの先例から、中核の設計原理を導く。**ガバナンスと研究は構造的に分離され、その間に慎重に設計されたインタフェースを置かなければならない**。

インタフェースは、競合する二要件を満たさなければならない。

1. **研究の独立**: ガバナンスは研究課題を統制できず、所見を抑圧できず、結論を改変できない。
2. **認識論的説明責任**: 研究は品質基準、すなわち健全な推論、証拠に基づく結論、反証可能な予測を満たさなければならない。

解は、*内容評価*と*品質評価*を分離することにある。Councilは論文の推論が妥当か（品質）を監査しうるが、論文の結論が都合よいか（内容）を監査してはならない。この区別は、査読者が方法論を評価し結果への同意を評価しない学術的ピアレビューから借用され、Arche憲法のArticle 10.3に明文化されている。

## 5. Akademiaアーキテクチャ (Akademia Architecture)

### 5.1 四役割構造

Akademiaは四つの専門役割からなり、それぞれが研究ライフサイクルの異なる相に責任を持つ。

| 役割 | 機能 | 相 | 歴史的類比 |
|------|----------|-------|-------------------|
| **Scholarch** | 研究方向、知的誠実性 | 方向づけ | 学部長／編集長 |
| **Scholar** | 外部知識の獲得 | 収集 | 研究司書／文献レビュアー |
| **Theorist** | 理論の展開と統合 | 統合 | 研究代表者／理論家 |
| **Scribe** | 論文化、公表、知識抽出 | 公表 | テクニカルライター／学術誌編集者 |

この四役割構造は、第3.3節で特定されたLogic Monopoly問題に直接対処する。単一の役割が研究ライフサイクル全体を支配しない。

- **Scholarch**は方向を定めるが、研究を実施したり論文を書いたりしない。
- **Scholar**は外部知識を収集するが、それを理論に統合しない。
- **Theorist**は理論を展開するが、研究方向を決めず、公表を支配しない。
- **Scribe**は構造化と公表を行うが、内容を決定しない。

この配分により、研究プロセスの各相が他相からのチェックに服する。不健全な理論を展開したTheoristは、内部レビューでScholarchに捕捉される。外部ソースを歪めたScholarは、Theoristが一貫しない所見を統合しようとするときに捕捉される。公表時に結論を歪めたScribeは、Theoristの統合との比較で捕捉される。

### 5.2 研究ワークフロー

研究ワークフローは六つの定義された相を経る。

1. **Collection** — Scholarが外部ソース（arXiv, web, 研究データベース）を検索し、所見を読み要約し、生の知識記事を知識基盤に書き込む。
2. **Synthesis** — Theoristが複数ソースを相互参照し、パターンと矛盾を特定し、仮説を立て、理論的枠組みを構築する。
3. **Drafting** — ScribeがTheoristの統合を論文形式に構造化し、構造的完全性と可読性を確保する。
4. **Internal Review** — ScholarchとTheoristが共同で内容の正確性と推論の誠実性を検証する。これはAkademia内部の主要な品質ゲートである。
5. **Quality Audit** — CouncilのQuality（Seat 4）とKnowledge（Seat 7）の座席が認識論的品質、すなわち推論の透明性、証拠の妥当性、構造的完全性を監査する。内容不一致に基づく監査は*してはならない*。
6. **Publication** — Scribeが論文を確定し、他のエージェントが消費するための運用知識を知識基盤へ抽出する。

### 5.3 研究トリガー

研究プログラムは、特定のトリガ条件が満たされたときにScholarchによって開始される。

- **重要な新しい外部研究**: ScholarがArcheの設計または運用に影響しうる公表物を特定する。
- **知識の臨界質量**: 知識基盤の相互参照密度が、複数の関連所見が統合準備完了であることを示す。
- **矛盾の検知**: システム内のいかなるエージェントも、知識記事間の矛盾をフラグし、調査を促す。
- **研究依頼**: CouncilまたはC-Suiteが特定課題の研究を依頼しうる。Scholarchはこれらを考慮するが、正当化とともに拒否もしうる。依頼は助言であり拘束的ではない。
- **論文改訂の必要性**: 新証拠により先行論文の結論の更新が必要になりうる。

### 5.4 独立のメカニズム

Akademiaの独立は慣行ではなく憲法規定（Article 10）によって保護される。

1. **報告関係なし**: AkademiaはCouncilまたはC-SuiteではなくFounderに報告する。
2. **内容への免疫**: Akademia外のいかなる役割も、論文の結論を改変し、遅延させ、または抑圧できない。
3. **課題の主権**: 研究課題はScholarchによって決まる。外部依頼は助言である。
4. **不服申立権**: AkademiaはCouncilの品質却下をFounderに申し立てうる。
5. **分離された説明責任の環**: Akademiaの役割はCouncilの説明責任の環から独立した閉じた環で相互評価する（Scholarch ← Theorist ← Scholar ← Scribe ← Scholarch）。

これらの保護は願望的ではなく構造的である。変更にはCouncilの準全会一致投票（12/13）とFounderの承認が要る憲法上の法として符号化されている。

## 6. ガバナンスとの関係 (Relationship to Governance)

### 6.1 品質と内容のファイアウォール

AkademiaとGovernance Councilの関係における最も重要な設計要素は、品質監査と内容統制の間のファイアウォールである。Article 10.3がこの区別を明文化する。

- **有効な品質異議**: 「この論文の推論連鎖は前提3と4の間に論理的ギャップがある。」（方法論を評価する。）
- **無効な内容異議**: 「我々のガバナンス機構に弱点があるというこの論文の結論は都合が悪い。」（所見の望ましさを評価する。）

区別は明示的テストによって運用化される。「この推論は欠陥がある」は有効な監査所見である。「この結論は都合が悪い」はそうではない。このテストは適用を意図的に単純にし、解釈的操作の表面積を減らす。

### 6.2 双方向の知識流

ガバナンスと研究は制度的に分離されているが、認識論的には接続されている。

**研究 → ガバナンス**: Akademiaは、ガバナンスの弱点の特定、構造改善の提案、またはガバナンス設計に関連する外部研究の統合などの所見を公表する。CouncilのEvolution座席（Seat 11）がこれらの所見の主要な消費者である。

**ガバナンス → 研究**: Councilの監査所見、Standing移行、異常フラグは、Akademiaの研究のための経験データを提供する。ガバナンスの失敗は研究機会である。各失敗は、Akademiaが調査し対処しうるシステム設計について何かを明かす。

**共有媒体としての知識基盤**: ガバナンスも研究も同一の知識基盤を読み書きする。知識基盤はインタフェース層として機能し、直接的な制度間コミュニケーションを要せずに研究所見が運用上アクセス可能であることを確保する。

### 6.3 モデル配分

Akademiaのモデル配分は、各役割の認知的要求を反映する。

| 役割 | モデル | 根拠 |
|------|-------|--------------|
| Scholarch | Default（最強） | 判断、方向設定、知的誠実性の評価を要する |
| Scholar | 分析はDefault、監視はfast | 深い分析には強い推論が要るが、定常監視には不要 |
| Theorist | Default（最強） | 統合、仮説形成、相互参照には最高の推論能力が要る |
| Scribe | 整形はfast、翻訳はdefault | 構造化作業は機械的だが、翻訳品質にはより強いモデルが要る |

この配分はModel Allocation Principle（Article 7）に従う。すなわち、戦略的／生成的作業には最強モデル、機械的／定常作業には最速モデルである。目標はシステム全体でトークン支出の90%超をfastモデルに置くことのままである。Akademiaの強モデルの偏った使用は、実行層のfastモデル専用使用によって釣り合わされる。

## 7. 研究による自己進化 (Self-Evolution Through Research)

### 7.1 知識—理論—枠組みの循環

Akademiaは、三段階の循環を通じてArcheの進化を可能にする。

**段階1: 知識の蓄積**。Scholarが外部ソースと内部システム挙動を継続的に監視し、所見を知識基盤に書き込む。時間とともに、知識基盤は何が機能し何が失敗し外部環境がどう見えるかについて豊かな経験記録を含むよう成長する。

**段階2: 理論の展開**。Theoristが蓄積知識を理論的枠組み、すなわち個々の所見を超えてパターンを特定し将来挙動を予測し構造改善を提案する説明モデルへ統合する。論文はこの段階の主要な出力である。

**段階3: 枠組みの改善**。公表された所見はCouncilのEvolution座席（Seat 11）に流入し、理論的洞察に基づいて規制または法の修正案を提案する。これらの改正は標準的投票プロセスを経るため、進化は研究に情報提供されつつガバナンスによって検証される。

この循環は自己強化である。枠組みの改善は新たなシステム挙動を生み、それが新たな経験データを生み、それが新たな知識蓄積を可能にし、それが新たな理論展開を可能にする。システムは外部再設計ではなく、内部の知識代謝を通じて進化する。

### 7.2 自己進化型エージェントパラダイムとの接続

知識—理論—枠組みの循環は、Self-Evolving Agents Survey（第3.1節）で特定された進化メカニズムに直接対応づけられる。

- **記憶進化**: 知識基盤はArcheの制度的記憶である。Akademiaの研究プロセスは、この記憶が進化するメカニズムである。単に蓄積するだけでなく、統合され、検証され、再構造化される。
- **アーキテクチャ進化**: Akademiaの所見が（Councilを通じて）憲法改正につながるとき、枠組みのアーキテクチャは進化する。これは統制された進化であり、一方的改変ではなくガバナンス投票によって媒介される。

第3.2節のSiriuSパラダイムは特に示唆に富む。SiriuSは、成功した推論軌跡を捕捉する経験ライブラリを通じて自己改善を達成する。Akademiaの知識基盤は類似の機能を果たす。事実だけでなく、行き止まりと訂正を含む推論連鎖を捕捉する（Article 5.3のAnti-Rationalization Protocolに従う）。将来のエージェントは、何が発見されたかだけでなく、どのように発見されたか、そして何が試され失敗したかから利益を得る。

### 7.3 Three Lawsの適用

Self-Evolving Agents SurveyのThree LawsをAkademiaに適用すると次のとおりである。

**Endure（安全性）**: 研究は憲法的整合性を損なってはならない。研究とガバナンスの分離によってこれが確保される。Akademiaは知識を生むが、規則を改正できるのはCouncilだけである。いかに洞察に富んでも、どの論文も枠組みの運用制約を直接変更しない。

**Excel（性能）**: 研究は運用上有用な知識を生まなければならない。Scribeの知識抽出相によってこれが確保される。すべての論文は、エージェントが直接消費しうる運用知識記事を結果として生む。最終的にシステム性能を改善しない研究は、使命に失敗した研究である。

**Evolve（最適化）**: 研究プロセス自体が時間とともに改善されなければならない。これを満たすのが最も難しい法則である。Akademiaは、累積的論文構築（各論文は先行研究を引用しその上に積む必要がある）、Corrections Log（先行論文の誤りが文書化されそこから学ばれる）、およびScholarchのメタ研究権限（研究プロセス自体についての研究を委託する能力）を通じてこれに対処する。

## 8. 限界と今後の課題 (Limitations and Future Work)

### 8.1 自動化のギャップ

Akademiaの設計は、AIエージェントが真の研究、すなわち意味のある問いの特定、証拠品質の評価、多様な所見の一貫した理論への統合、限界の正直な報告を実行しうることを前提とする。現在の大規模言語モデルはこれらの機能を近似しうるが、完全には再現できない。Akademiaの出力の質は、役割を具現化するモデルの推論能力によって上限が決まる。

**緩和**: ScholarchとTheoristの役割には、推論の質が拘束条件となるため、利用可能な最強モデルが割り当てられる。モデル能力が向上すれば、構造変更なしにAkademiaの出力品質が向上する。アーキテクチャはモデル能力に応じてスケールするよう設計されている。

### 8.2 外部ソースの検証

Scholarの役割は外部研究を収集するが、現行システムには外部ソースの真正性または再現可能性を検証するメカニズムはない。捏造されたarXivプレプリント、撤回前に引用された撤回済み論文、または意図的に誤解を招く要約は、汚染された知識をシステムに導入しうる。

**緩和**: Theoristの統合相が部分的なチェックを提供する。他のソースと矛盾する捏造所見は検知可能な矛盾を生む。しかし、既存知識と内部的に一貫した洗練された捏造は捕捉されない。将来の研究では、自動引用検証と複製チェックを探るべきである。

### 8.3 研究の優先順位付け

Scholarchは研究課題について単独の権限を持ち、研究方向における単一障害点を生む。系統的に偏った優先順位のScholarchは、Akademiaを重大な問いを無視しながら無関係な問いを調査する方向へ導きうる。

**緩和**: ScholarchのStandingはScribeによって評価され（相互説明責任の環）、CouncilのKnowledge座は知識基盤全体の品質を監視する。持続的な研究の無関係性はStandingの低下を引き起こす。しかし、もっともらしいが低優先度の課題を研究するという微妙な誤誘導は、検知がより難しい。

### 8.4 知識基盤の規模

知識基盤が成長するにつれ、「作業前に読む」義務（Article 5.1）のコストが増す。エージェントはいかなるタスクを始める前にもますます大きな知識貯蔵を検索しなければならない。効果的な知識組織化と検索がなければ、知識基盤は資産から負債へ移行する。

**緩和**: Curatorの統合と剪定の機能がこれを部分的に緩和する。将来の研究では、階層的知識組織化、関連性に基づく検索、自動知識グラフ構築を探るべきである。

### 8.5 ワークスペース横断の研究一貫性

Akademiaはグローバル（Layer 0）で動作するが、その研究はワークスペース間で関連度が異なりうる。ウェブセキュリティ実践に関する所見はウェブアプリケーションのワークスペースに高度に関連し、データ分析のワークスペースには無関係である。現行設計にはワークスペース固有の研究優先順位付けのメカニズムは含まれない。

### 8.6 負の結果の正直な報告

Anti-Rationalization Protocol（Article 5.3）は行き止まりと却下された代替案の文書化を要求する。しかし、印象的な研究出力を生むインセンティブと、調査が実行可能な所見を生まなかったときに正直に報告する義務との間には内在的張力がある。現行設計はProtocolの要件と相互説明責任に依存して誠実性を維持するが、インセンティブ構造は強化されうる。

## 9. 結論 (Conclusions)

Akademiaは、憲法的AIガバナンスにおける構造的隙間、すなわちシステム進化に必要な新知識を監視メカニズムが生成できないことに対処する。分析はいくつかの結論へ至る。

**知識の劣化は能動的研究なしには不可避である。** 静的知識基盤は時間的・文脈的・推論的軸に沿って劣化する。ガバナンスは現在の知識への適合は強制できるが、その知識が不十分になったことは検知できない。専用の研究機関は贅沢ではなく構造的要件である。

**研究の独立は望ましいだけでなく構造的に必須である。** 国家統制研究の歴史は、現在の規則に制度的利害を持つガバナンス機関が、それらの規則を脅かす研究を体系的に歪めることを示す。品質と内容のファイアウォール（第6.1節）がこの分離を運用化する。

**Logic Monopoly問題は研究にも当てはまる。** 問いの特定、調査、所見の統合、品質評価を単一実体が行うとき、それはLogic Monopoly文献が行動エージェントで特定するのと同種の推論の集中脆弱性を示す。Akademiaの四役割構造は、この集中を防ぐために研究ライフサイクルを分散する。

**自己進化には制度的インフラが要る。** Self-Evolving Agents SurveyとSiriuSは、構造化された知識蓄積を通じてAIシステムが改善しうることを示す。Akademiaはこの蓄積のための制度的インフラを提供する。アドホックなプロセスとしてではなく、定義された役割・ワークフロー・品質基準を伴う憲法上保護された機能としてである。

**知識—理論—枠組みの循環は統制された進化を可能にする。** 知識生成（Akademia）と規則変更（Council）を分離することにより、Archeは、研究が構造変更を直接決定せず情報提供する媒介プロセスを通じて進化する。これはEndureの法則を満たす。すなわち、憲法的整合性を損なわずに進化が起きる。

Akademiaは究極的には制度的賭けである。AIガバナンス枠組みの知識基盤の質が長期的有効性の主要な決定因であり、高品質な知識の生産には、人類が千年の制度実験を通じて発見してきたのと同じ構造的保護、すなわち独立・説明責任・専門化・累積的構築が要る、という賭けである。

## 付録: 反証可能な予測 (Appendix: Falsifiable Predictions)

1. **知識基盤の陳腐化率**: Akademiaの研究プロセスが活動的なワークスペースでは、知識記事の陳腐化率（Curatorの監査が時代遅れの内容をフラグする頻度として測定）は、能動的研究プロセスのないワークスペースに比べて、10セッションの窓で少なくとも40%低い。

2. **枠組み進化の頻度**: 能動的Akademiaを持つワークスペースは、10セッションあたりの証拠に基づく規制修正案の提案（CouncilのEvolution座席経由）を、アドホックな知識管理のみに依存するワークスペースに比べて少なくとも2倍多く生む。

3. **Logic Monopoly耐性**: 四役割パイプライン（Scholarch → Scholar → Theorist → Scribe）で生産された研究出力は、四機能すべてを単一エージェントが行う研究に比べて、論文あたりの推論誤りが少ない。Councilの品質監査却下率で測定する。予測閾値: 品質却下は少なくとも30%減少。

4. **品質と内容のファイアウォールの健全性**: 20回超のCouncilによるAkademia論文の品質監査のうち、監査異議の10%未満が内容に基づく（無効）ものではなく品質に基づく（有効）ものとして分類され、ファイアウォールの区別が運用上維持可能であることを示す。

5. **相互説明責任の検知率**: Akademia内部の説明責任の環（Scholarch ← Theorist ← Scholar ← Scribe ← Scholarch）において、能動研究の10セッションあたり少なくとも一つのStandingに関わる品質問題が検知され、環がゴム印ではなく能動的品質メカニズムとして機能していることを示す。

6. **累積知識効果**: 五番目のAkademia論文以降に公表される論文は、平均して少なくとも二つの先行Akademia論文を引用し、累積的構築要件が孤立した調査ではなく真の知的足場を生んでいることを示す。

7. **外部研究統合の遅延**: 有意な外部研究所見（Archeの設計または運用に直接関連する論文として定義）は、Scholarが特定してから3セッション以内に知識基盤へ統合され、研究パイプラインが環境変化に適合可能なペースで動作していることを示す。
